Reframing · 1 of 202 / A Common Mistake
The question everyone asks first

"Which AI model is best?"

This is the wrong question.
Reaching for a bigger model treats RAG as a black box that gets smarter on its own. But on the Annals, a better model does not fix a missing source, a badly cut chunk, or an encoder that cannot read old Korean. Those are set before the model runs.
Reframing · 2 of 203 / The Right Frame
A better question to ask

RAG is not an analysis tool.
It is a research interface.

not a machine that produces answers you accept.
but an environment that helps you find, compare, and return to sources.
So whether RAG works on the Annals depends far less on the model —
and far more on the design choices we make before the model ever runs:
how we prepare the data, how we retrieve, and how we check the result.

① Source Foundation07 / The Encoder
The model that turns text into vectors

A general encoder does not know old Korean

Why a general encoder struggles
the problemGeneral encoders learn modern language. In the Annals, the same person, place, or office is written differently across periods and sources — and a modern question rarely shares words with the classical record.
what this means
Classical Chinese, idu, and archaic names sit far from a modern query in a general encoder's vector space. The right record can be there — and still never surface.
Distance in the model is not the same as historical relevance. So how do we fix it? →
① Source Foundation08 / The Encoder
Teaching the model what "similar" means

Fine-tune it — and the historian decides

After fine-tuning on historical sources
the fixFine-tune the encoder on historical sources. One study trained on the Annals + Seungjeongwon Ilgi, then retrieved related records that shared no words with the query.
key distinctionStoring text in a vector DB is not the same as training the encoder. The DB just fills up; the encoder's sense of "similar" does not change.
The historian's role
You decide which historical relations count as "similar" — an alias, a variant spelling, a changed institution. The encoder measures distance; it does not settle relevance.
② Procedural Auditability09 / Query Rewriting
The system rewrites your question

You should be able to see how

1 · what the user types
"Was there an office to hunt down escaped slaves under King Hyojong?"
↓
2 · checked against known rules
expand termescaped slaves → 推刷 · office that recovered runaways
alias / variantthe office → 推刷都監 (Chuswae-dogam)
estimate periodKing Hyojong → reign 1649–1659
↓
3 · rewritten query sent to search
"Records on the 推刷都監 that recovered escaped slaves, active in the reign of King Hyojong (1649–1659)."
The point: rewriting quietly includes and excludes records. If the added terms, aliases, and date filters are logged, a historian can check what the system decided — and why a record was found or missed.

③ Evaluative Feedback12 / Human-in-the-loop
Evaluation is the start of the next run

The historian is not the user — the designer

Score retrieval and generation separately — so you know which stage failed. But a number cannot tell you if a source was the right kind of evidence. That judgment is historical, and it must loop back into the data, the retrieval, and the prompt.

Takeaway16 / Close

RAG will not replace
source criticism.

  • It finds, compares, and provisionally synthesizes — then hands you back to the original text.
  • Feasibility rests on design, not the model: data, chunking, encoder, retrieval, evaluation.
  • The historian is not the end user. The historian is the designer of the research environment.
